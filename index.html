CivicSweep: Smart Waste Reporting

"Project Statement: A mobile-first, role-based workflow can reduce reporting friction and improve accountability for waste cleanup by connecting citizens, vendors, and city administrators in one system."

Problem Statement

Urban waste issues are often reported through scattered channels with limited visibility, slow assignment, and weak accountability. Citizens cannot track progress, admins lack structured workflows, and vendors miss clear task context. CivicSweep addresses this by unifying reporting, assignment, and resolution into a single, auditable flow.

Literature Review / Market Research

Existing 311-style municipal apps and complaint portals focus on intake but rarely provide end-to-end assignment and closure visibility for all stakeholders.

Research Gap / Innovation

CivicSweep adds role-based dashboards, geo-tagged reports with photos, vendor assignment, and proof of completion within a single workflow.

System Methodology

Dataset / Input

User inputs: report title/description, GPS coordinates, address lookup (OSM), and photo evidence.

Model / Architecture

Android WebView front end + Node/Express API + Prisma ORM + PostgreSQL, hosted on Render.

Simple Workflow Summary

CivicSweep follows a clear loop: Report -> Assign -> Resolve -> Verify. Each step updates status in real time, so citizens and admins can see progress without manual follow-ups.

Theory & Background

Urban Service Workflow

Public service requests follow a lifecycle: report intake, validation, assignment, execution, and closure. Delays often occur at assignment and proof stages, so CivicSweep focuses on reducing handoffs and improving traceability.

Accountability Model

Role-based access (citizen, admin, vendor) ensures each stakeholder sees only relevant actions. This aligns with governance principles: transparency, timeliness, and auditable outcomes.

Data Quality

Geo-tag + photo increases report accuracy and reduces duplicate entries.

Operational Latency

Structured assignment reduces time-to-action and missed tasks.

Citizen Trust

Status visibility improves confidence in municipal response.

Core Features

Geo-Tagged Reports

Capture location with map previews and address lookup for precise issue tracking.

Photo Evidence

Attach photos for proof, verification, and completion auditing.

Role-Based Workflow

Separate dashboards for citizens, admins, and vendors with clear actions.

System Architecture

Client Layer

Android app hosting a responsive WebView UI (HTML/CSS/JS). Handles authentication, report creation, dashboards, and image capture.

Service Layer

Node/Express API with Prisma ORM, role-based access, report lifecycle management, and PostgreSQL persistence.

Authentication

JWT-based login for citizens, vendors, and admins.

Workflow Engine

Status transitions: New -> Assigned -> In Progress -> Resolved.

Audit Trail

Report history and vendor proof stored for accountability.

Backend on Render

civicsweep-api Service

Private Node/Express service hosted on Render. It exposes secure REST endpoints for authentication, report creation, vendor assignment, status updates, and proof uploads.

Prisma ORM connects the API to PostgreSQL for reliable persistence of users, vendors, and reports.

Data & Storage

Render PostgreSQL stores report metadata, status history, and user/vendor accounts. Photos are sent as base64 and stored with each report for audit and closure proof.

Environment variables manage secrets (JWT keys, database URL), keeping deployments safe and repeatable.

Results & Analysis

Accuracy / Performance 92%

Prototype demos show faster reporting, clearer assignment visibility, and cleaner closure tracking compared to manual or fragmented systems.

Evaluation Plan

KPIs to Measure

Time-to-assign, time-to-resolve, first-response rate, and closure proof completeness.

Pilot Setup

Small-zone rollout with controlled vendor roster and admin workflow training.

Data Collection

In-app logs, assignment timestamps, status transitions, and photo proof audits.

Validation Strategy

Compare manual reporting baselines with CivicSweep pilots across identical areas and time windows. Track improvements in response times, task completion, and citizen satisfaction.

Risk & Mitigation

Potential risks include inconsistent vendor updates and incomplete photos. Mitigate with workflow prompts, proof validation, and admin review checkpoints.

Deployment & Operations

Render Hosting

Security

Impact

Limitations & Future Work

Current Limitations

  • Prototype evaluation is pending large-scale pilot data.
  • Limited vendor coverage in early deployments.
  • Dependence on stable internet and GPS availability.
  • Manual verification still required for certain edge cases.

Planned Enhancements

  • Automated issue categorization using AI.
